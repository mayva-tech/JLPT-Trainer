## 1. Product overview

A calm, full-screen browser slideshow that presents one JLPT N2 vocabulary item at a time and plays pre-recorded, reviewed audio in stages (word → reading → meaning → phrase → phrase again). You run it locally with `npm run dev`, open it in Edge/Chrome, go full-screen, and screen-record it into a 16:9 video for YouTube. The same video serves both watchers (kanji + meaning on screen) and audio-only listeners walking or commuting.

## 2. Exact MVP scope

**In:** local Vite dev server; a fixed 16:9 stage that scales proportionally to any window size; 1 lesson of 10 fully-detailed N2 items; progressive text reveal synced to audio; the player sequence with app-controlled pauses; the five study modes (Full / Japanese-Only / Recall / Phrase Practice / Audio-First); on-screen controls that hide for recording; keyboard shortcuts; optional intro/outro cards; a recording-safe mode; graceful "audio unavailable" handling; and a Node validation script.

**Out:** everything you listed — no backend, accounts, database, ads, cloud, browser TTS, or any mobile/Expo code.

**Reused from before:** the 10 curated, human-checked vocabulary entries (readings, kanji breakdowns, natural short phrases). Audio references change from bundled `require()` numbers to simple `/audio/...` string URLs, which suits the web.

## 3. Windows recording workflow

Run `npm run dev` → browser opens the app → pick the lesson → press **F** for full-screen and **H** to hide controls (recording-safe mode locks the frame to 16:9). Record the window with **OBS Studio** (set a 1920×1080 canvas and a Window Capture on the browser) or CapCut/Snipping Tool. Press Space to start; playback auto-advances through all 10 items and stops on the outro card. Stop the recording, export, upload. No internet needed after install.

A web convenience worth knowing: unlike the mobile bundler, Vite does **not** break if an audio file is missing — the `public/` folder is served as-is and a missing `.mp3` simply fails at runtime, where the app catches it and shows "Audio unavailable" without crashing. So you can build and run today with zero audio files, then drop clips in later with no code changes.

## 4. Screen architecture

A single-page app with two views: a **Lesson Selection** page and a **Player** page. The Player renders a scaling 16:9 **stage** containing whichever card is active — intro, a `VocabularySlide`, or outro — plus a subtle progress indicator and a hideable control panel. All sequencing/audio logic lives in `services/` and a `useLessonPlayer` hook; the visual components stay dumb and just render the current stage. This keeps presentation and playback cleanly separated, as required.

## 5. Playback sequence

Per item: word audio → short pause → reading audio → short pause → meaning audio → recall pause → phrase audio → short pause → phrase audio again → final display pause → transition to next. Text reveals one line at a time as each clip plays. Pauses are `setTimeout` gaps owned by the player (never embedded in files) and driven by the config object. Only one clip ever plays at once — starting any new clip stops the previous one first.

## 6. Data models

I'll use your models verbatim (`VocabularyItem` with string `VocabularyAudio` paths, `KanjiDetail`, and the separate `Lesson` type with `vocabularyIds`), plus a small `player.ts` types file for `StudyMode` and the reveal stages.

## 7. Folder structure

Exactly your suggested tree (`public/audio/...`, `src/{components,data,hooks,services,types,config,pages,styles}`, `scripts/validate-audio-files.mjs`). Stage 1 creates the types, config, data, styles, one component, the progress indicator, and the Player page; the rest arrive in later stages.

## 8. Implementation roadmap

- **Stage 1 (this response):** Vite setup, models, 10-item data + lesson, the scaling 16:9 layout, and basic manual prev/next navigation.
- **Stage 2:** audio service, sequence builder, `useLessonPlayer`, automatic playback, progressive reveal, controls, keyboard shortcuts.
- **Stage 3:** full-screen recording mode, intro/outro cards, lesson selection, missing-audio handling, validation script.
- **Stage 4:** visual polish, timing controls UI, the remaining study modes, final recording instructions.

```tsx
  }

  const item = items[index];
  const goPrev = () => setIndex((i) => Math.max(0, i - 1));
  const goNext = () => setIndex((i) => Math.min(items.length - 1, i + 1));

  return (
    <>
      <div className="stage-wrapper">
        <div className="stage">
          <div className="lesson-title">{lesson.title}</div>
          <ProgressIndicator current={index} total={items.length} />
          {item && <VocabularySlide item={item} reveal={FULLY_REVEALED} />}
        </div>
      </div>

      <div className="nav-bar">
        <button onClick={goPrev} disabled={index === 0}>
          ← Previous
        </button>
        <button onClick={goNext} disabled={index === items.length - 1}>
          Next →
        </button>
      </div>
    </>
  );
}
```
